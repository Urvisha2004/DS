SELECT 
   (1 - (phy.value / (cur.value + con.value))) * 100 AS buffer_cache_hit_ratio
FROM 
   v$sysstat cur,
   v$sysstat con,
   v$sysstat phy
WHERE 
   cur.name = 'db block gets'
   AND con.name = 'consistent gets'
   AND phy.name = 'physical reads';

-------------------------------
current mode
consistent mode
physical reads
---------------
formula:-
(1 - (phy.value / (cur.value + con.value))) * 100
---------------
ex:-

db block gets = 1,000,000
consistent gets = 4,000,000
physical reads = 200,000

1,000,000 + 4,000,000 = 5,000,000

200,000 / 5,000,000 = 0.04  (4%)

1 - 0.04 = 0.96  (96%)
0.96 * 100 = 96%

------------
≥ 90% → Excellent (database is using memory effectively).
80–90% → Acceptable, but could improve.
< 80% → Poor, too many disk reads (bad performance).
